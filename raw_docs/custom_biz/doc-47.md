## 意图

意图指一个明确的用户目的。`例：餐厅技能包含订位、点餐意图`
意图是技能的基础组成部分，每个意图一般代表一个或一组功能。

### 1.1. 意图分类

- **入口意图**：不需要历史信息就响应用户
- **对话意图**：为了减少不同技能的说法串扰，先命中A技能的入口意图，才能命中A技能的对话意图。存在历史交互信息，响应存在上下文关联。

示例：

```text
// 正常命中
用户：今天天气   （入口意图）
AIUI：晴
用户：明天呢     （对话意图）
AIUI：阴天
```

```text
// 无法命中
用户：明天呢     （对话意图）
AIUI：对不起，我没听懂 （未命中）
```

「明天呢」没有单独存在的意义，系统不响应。只有具备历史信息后，对话意图才响应用户。

## 语料

意图一般是一类语料的集合，以实现某个或某组功能。用户说法多样，技能应尽可能完善语料，否则系统无法理解。语料由槽位及其他文本内容构成，每个槽位都可绑定一个实体、辅助词或通配实体（特殊实体）等元素，相关介绍可见[实体和辅助词](https://aiui-doc.xf-yun.com/project-1/doc-48/ "实体和辅助词")。
示例：

- 今天天气
- 今天下雨吗
- 告诉我今天天气
- ……

### 2.1. 语料分类

**模版语料（推荐）**：由`{词槽}`和文本构成，`{}`内为槽位，每个词槽都可以绑定实体或辅助词。 `例：{time}天气`
**例句语料**：由纯文本构成，可用鼠标选择其中的词来配置词槽和绑定实体、辅助词。 `例：今天天气`

**例句语料和模板语料无法在一个意图中共存，会导致无法正常命中。**

### 2.2. 语料编写

模版语料仅支持汉字、数字、字母及`<>`、`{}`、`[]`、`()`、`,`、`。`，`.`、`!`、`?`、`%`、`|`、`+`、`-`、`×`、`*`、`÷`、`/`、`_`、`#`、`'`、`"`、`°`、`=`。

例句语料仅支持`,`、`。`、`.`、`!`、`?`、`%`、`+`、`-`、`×`、`*`、`÷`、`/`、`_`、`#`、`'`、`"`、`°`、`=`。

### 2.3. 标签符号说明

`{}`：代表词槽，内容仅支持：字母、数字、下划线`_`、小数点`.`、井号`#`。输入该符号，则语料强制转为模板语料，成对出现。例：{time}天气

`()`：代表必选，内容支持词槽、汉字等。输入后语料强制转为模板语料，每句语料最多配置5对。 `例:（今天|今天的）天气`等同于

- 今天天气
- 今天的天气

`|`：代表或，仅推荐出现在`()`中

`[]`：代表内容可有可无，包含词槽、汉字、字母等。输入后语料强制转为模板语料，每句语料最多配置5对。 例:[今天][今天的]天气 等同于

- 天气
- 今天天气
- 今天的天气
  > *例句语料如需配置词槽可有可无，可将对应词槽转换为辅助词，并取消勾选表述必须*。
