title: 长时记忆配置
---

::: info 概述
长时记忆能力让 AIUI 突破对话上下文窗口限制，跨会话持久保存用户画像、偏好和对话历史，让设备真正"记得"用户，提供更自然、更个性化的长期交互体验。

**使用流程**：

1. 登录 [AIUI 平台](https://aiui.xfyun.cn/) →「我的应用」
2. 选择目标应用 →「应用配置」
3. 找到「长时记忆」配置项并开启
:::

## 背景

在普通的对话系统中，模型仅依赖“上下文窗口”来维持对话。一旦超出对话轮数限制或会话结束，对话信息就会丢失，导致AIUI变成“只有7秒记忆的金鱼”。

## 长时记忆介绍

**`长时记忆`** 是指系统具备跨会话、持久化存储并检索用户关键信息的一种大模型原子能力。它允许 AIUI 像人类朋友一样，记住用户几天、几周甚至几个月前提到过的偏好、事实和经历，并在后续的对话中自然地调用这些信息。
长时记忆的价值：

- 连续对话。打破单次会话的隔离，建立连贯的交互历史。
- 个性化回复。基于历史数据构建用户画像，提供千人千面的对话服务。
- 情感链接。 记住用户的偏好、特定要求，持续迭代交互表现，显著提升用户的信任感。

## 功能现状与规划

目前AIUI已具备完整的“自动记忆闭环”，用户无需干预即可体验记忆带来的连续性。

- **智能记忆采集**
   系统自动识别对话中的关键信息（用户画像、对话记录），采用大模型进行自主决策信息，对原始对话记录进行信息抽取与信息压缩。

  `AIUI记忆特征`表现为：

  - 根据对话内容判定重要性，对用户有意义的信息更不容易被遗忘。
  - 最近的内容比过往的内容更详细，老旧的记忆容易丢失细节。
  - 记忆发生冲突，新记忆覆盖旧记忆；相似的信息，新旧记忆会合并。

  `场景示例`：用户提到“我和同学小胖经常去公园玩”，系统自动捕获隐藏的社会关系“小胖是用户的同学”并存储。
- **动态记忆调用**
   在新对话中，系统会自动检索历史记忆，并将其注入到对话上下文中。

  - 记忆的归属：记忆打通声纹，通过声纹注册，记忆将在用户层级生效，不同用户的记忆会隔离存储。如果没有开启声纹配置，记忆将在设备层级生效，所有交互用户共享一份记忆。
  - 检索时间范围：当前系统仅支持最近7天的对话内容的回溯，用户画像相关的个人基础信息、偏好等则支持全量回溯。
  - 长短记忆配合：对话历史支持10轮，是交互系统的短时记忆。对话每满10轮或停止交互10分钟后触发一次记忆采集，短时记忆转换为长时记忆。长短记忆需配合使用来保障模型不会失忆，因此在调用AIUI时需将new\_session字段设置为false[详见极速API协议文档](/api-dev/ultra-chain/interact-api "详见极速API协议文档")，否则可能出现记忆丢失现象。
- **全局开关控制**
   开发者可在应用情景下配置，使用该情景服务的用户是否开启长时记忆能力。注意仅支持选用`交互大模型v2`、`儿童交互大模型`时可调用长时记忆能力。

## 使用方式

温馨提示

长时记忆服务能力仅在AIUI极速超拟人交互链路下支持。

- 联系讯飞商务申请获取能力授权
- 授权通过后，在AIUI极速类应用管理页面 **语义模型配置 - 个性化设置** 下直接勾选开启即可。
