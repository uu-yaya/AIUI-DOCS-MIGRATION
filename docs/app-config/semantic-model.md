语义模型配置概述

语义模型配置是AIUI极速超拟人交互类应用提供的配置项目，主要给开发者提供传统语义技能（官方技能、问答库）、大模型语义功能模块（文档问答库、智能体、联网搜索）相关能力配置选项。

平台分类如下：

**- 添加工具**

**- 搜索设置**

## 1.添加工具

### 1.1. 问答

该配置项是给极速链路配置自定义大模型链路的**文档问答**。
点击“添加”，可以配置当前账号下创建并发布好的文档问答，另外开发者也可以配置文档问答在服务链路的集成策略【针对所有文档问答生效】：

- 检索方法
  - 单向量召回
  - 多路召回
- 相关性阈值

知识说明

1、相关性就是知识片段和用户问题的匹配程度，数值在 0 到 1 之间。这个值越高，说明这个知识片段越有可能帮你答好问题。

2、检索方法就是计算相关性的方法。单向量召回是靠算语义向量的相似度来得出结果；多路召回则是把语义向量、关键词等多种方式的相似度合起来计算。

为了筛选出足够匹配的知识片段，会设定一个 “相关性阈值”—— 比如阈值设为 0.6，就意味着只保留相关性数值≥0.6 的知识片段，低于这个值的会被排除，避免不相关的知识干扰回答。

### 1.2. 智能体

该配置项是给极速链路配置自定义大模型链路的**新版智能体**。
点击“添加”，可以配置当前账号下创建并发布好的新版智能体，点击“创建智能体”也可以跳转到智能体开发界面进行自定义智能体开发。

知识说明

1、旧版智能体：仅提供意图匹配能力，开发者编辑时仅提供语料设置

2、新版智能体：除提供意图匹配能力，开发者可以按照业务需要进行节点能力模块自定义编辑。与目前主流智能体开发方式基本保持一致

### 1.3. 技能

该配置项是给极速链路配置传统语义技能能力，支持类型有：

- 官方技能
- 自定义技能
- 语句问答
- 关键词问答

并且还支持进行特性设置，如自定义技能或自定义语句问答的阈值设置、关键词过滤、语义敏感词、精简协议

#### 1.3.1. 技能配置

点击“添加”，可在当前应用的情景模式下配置需要的技能。

![示例图片](/media/202508/2025-08-12_192349_5810370.3654981502618089.png)

#### 1.3.2. 自定义技能阈值设置

在自定义技能添加界面，勾选自定义技能后，点击技能下的设置图标进行设置，注意支持的技能类型：
**自定义私有技能、自定义语句问答**
自定义阈值主要是为了做技能匹配度调节的，可以根据实际需要(模糊匹配度和误识别两个维度)综合考虑设置。
AIUI技能实际触发的模糊匹配程度，可以查看技能json中的source字段取值，可以参考该取值进行自定义技能阈值调整。

- 技能阈值越高：技能对出发的语句格式要求越严（模糊匹配越低，不容易误识别）
- 技能阈值越低：技能对出发的语句格式要求越送（模糊匹配越高，容易误识别）

#### 1.3.3. 技能设置

技能配置后，还可以点击“设置”进行一些高级能力配置，开发者按需设置。

- **关键词过滤**
  过滤识别结果中的关键词，不进行语义理解。常用于过滤唤醒词
   比如：唤醒词是“小飞小飞”
   **过滤前**：“小飞小飞，你好“
   **过滤后**：“你好“
- **语义敏感词过滤**
  给用户回复的信息中包含敏感信息，可以通过设置过滤掉。支持自定义增加。
  注意：
  到触发语义敏感词过滤时，固定响应：

  ```
  {
    answer: {
        text: "文明社会，请文明用语。",
        topic: "",
        type: "T"
    },
    operation: "ANSWER",
    rc: 0,
    service: "forbidden",
    sid: "xxxxxxxxx",
    state: null,
    text: "xxx",
    uuid: "xxxxxxxxx"
  }
  ```

  特别说明：极速交互链路本身在服务层面各个模组的输入输出都是有单独的安全审核服务进行监控的。
- **精简协议**
  简化结构化语义，将技能结果分为：媒资类和播报类。默认关闭。
  将多个技能精简成一种输出协议，大大降低技能适配工作。适用于无屏设备。

## 2.搜索设置

### 2.1. 搜索引擎

该项提供大模型联网搜索能力项，默认是关闭的，开发者可以自行进行开启。
当开启后用户进行语音交互时，当服务判断需要进行搜索时，会调用联网服务进行信息搜索，然后大模型对搜索的信息进行总结答复。

![](/media/202508/2025-08-12_193821_4874020.12752165493357404.png "null")

### 2.2. 知识溯源

该能力项主要是控制大模型服务结果下发的，通常作为服务链路中加载文档问答或联网搜索获取的知识点，内容比较多，默认是不下发的。当开发者需要了解大模型回复的内容从哪里获取的时候，可以开启该能力项，开启后AIUI服务在交互时，会新增下发 cbm\_knowledge 结果。
