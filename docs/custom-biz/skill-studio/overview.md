## 技能工作室简介

技能工作室包含**自定义技能**、**语句问答**、**关键词问答**、**设备人设**，**官方技能**。
AIUI平台支持开发者免费开发私有技能、问答库、文档问答（大模型链路）。

## 技能配置说明

AIUI应用开启结构化语义后，可进行技能任意搭配设置。
按粒度划分：

- 1个应用（设备）包含多个技能
- 1个技能包含多个意图
- 1个意图包含多个语料

## 私有技能开发步骤

- 设计交互流程

  先梳理客户或业务交互需求，整理后画一个流程图，描述用户请求和期望的结果；把请求抽象成[意图](/custom-biz/skill-studio/intents)。可参考[技能设计规范](/custom-biz/skill-studio/voice-design-guide)。
- 选择合适的实现方式
   根据梳理业务方式选择合适的实现方式：自定义技能 、问答库，详细分类说明如下

  - 自定义技能-**私有技能**：
    基于客户端私有业务进行流程业务编辑，能力强大，可实现复杂业务处理。技能构建发布无需讯飞审核，编写的技能发布后可在当前账号下所有应用下使用。
  - 自定义技能-**开放技能**：
    基于公开业务或创新业务实现技能开发，可实现复杂业务。该类型技能需要通过讯飞内部审核，通过后开放到官方技能类型下，可对AIUI所有用户开放。
  - 自定义技能-**定制技能**：
    当讯飞官方技能客户侧需要集成使用，但又有需求差异，如部分能力不支持或说法与实际需求有差异，可以基于官方技能做定制技能开发。
  - 自定义问答库-**语句问答**：
    当交互内容和回复都是明确的，如QA问答，可以直接创建语句问答库来实现。
  - 自定义问答库-**关键词问答**：
    当问题回复内容相对固定，问题查询需要做关键词匹配时，可以创建关键词问答实现。
  - 自定义问答库-**文档问答**：
    这是大模型提供的能力，当本地有相关知识文档，希望讯飞能基于这些文档做知识提取并满足交互问答，可以创建文档。该能力使用时需要开启交互大模型。
- 在控制台创建技能
  以简单的私有技能创建流程做演示，具体步操作可以[查看指引](/custom-biz/skill-studio/development/create)
- 完善技能的交互模型
   将第1步的意图配置在技能中。完善用户语料。[查看指引](/custom-biz/skill-studio/development/intent-config)
- 技能逻辑处理
   当语义理解识别意图后，你需要编码来满足用户的意图，支持 **在线云函数** 和 **webhook** 。当用户交互时，你的技能会收到讯飞服务器发送给你的 `Request`，你在接到`Request`之后进行处理并返回`Response`，讯飞将会根据返回的`Response`进行后续处理。[查看指引](/custom-biz/skill-studio/development/post-process)

  使用 v2.1协议，查看[2.1请求](/custom-biz/protocols/post-process-request-v21)和[2.1回复](/custom-biz/protocols/post-process-response-v21)。
- 测试技能

  技能发布前需测试并修复缺陷，[查看指引](/custom-biz/skill-studio/development/testing)
- 发布技能

  私有技能可直接发布，并在应用中配置。[查看指引](/custom-biz/skill-studio/development/publish)
