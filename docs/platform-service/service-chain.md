::: info 概述
AIUI平台针对语义交互场景，目前提供三种服务链路，按照服务能力支持的时间先后顺序分别为：
:::

## 传统语义交互链路

传统语义交互链路是AIUI平台2016底开始推出的，以传统语义理解能力为核心，为人机交互方案提供结构化服务结果。
其中结构化语义理解主要采用语法句式匹配的技术方案，其支持配置的能力项有：

- 商店技能
- 自定义技能
- 语句问答
- 关键词问答
- 设备人设
- 闲聊兜底

### 1.1. 应用配置

在老AIUI应用配置中开启结构化语义

温馨提示

传统语义链路不可以开启大模型配置。关于新老应用说明，见文档[【AIUI应用介绍】](https://aiui-doc-admin.xf-yun.com/project-14/doc-604/)说明。

![](/media/202507/2025-07-29_163839_0376990.0007209118432673911.png)

### 1.2. 流程说明

传统语义链路云服务支持能力有：语音识别、结构化语义、应用后处理、语音合成能力，其全能力服务流程为：

**语音识别 → 结构化语义 → 后处理 → 语音合成**

该流程顺序固定，除语音识别外的其他模块可开关，且配置修改时按照整体流程插入或移除处理项。
下面为AIUI传统语义链路全能力适配的服务流程图：

![](/media/202508/2025-08-21_192059_9358860.5911447368528479.png)

如上图所示，云端开启后处理+语音合成全链路能力时，后处理服务接受AIUI云服务转发的语义结果做业务处理，然后返回结果给到AIUI,然后语音合成服务从接受到的后处理结果中提取合成文本做合成处理。

**技能优先级说明：**

- 自定义技能 > 语句问答 > 设备人设 > 商店技能 > 关键词问答 > 兜底闲聊
- 同类型技能不区分优先级。

::: warning 注意
全链路使用时，语音合成服务是从后处理结果的 answer.text 字段中提取合成文本的。所以在配置应用后处理+语音合成场景下，后处理返回的结果需要满足下面的格式：
:::

{"intent" : { "answer" : {"text":"xxxx"}}}

## 通用大模型交互链路

### 2.1. 应用配置

传统语义和大模型语义主要区别在于是否开启“大模型”。开启大模型配置后，此时应用服务配置指向到通用大模型链路授权。

![](/media/202507/2025-07-29_190539_7844830.6991171186085761.png)

::: warning 注意
老应用未进行授权时，可以任意切换服务链路。但是当应用有过传统语义链路授权后，大模型能力将不再支持随意开关。关闭后再开启会提示“您的应用为正式授权过通用语义模型的应用，若想使用交互大模型Lite版请联系商务。”
:::

![](/media/202507/2025-07-29_191303_9113110.9890031402784571.png)

### 2.2. 流程说明

通用大模型交互链路不是仅仅在传统语义交互链路后面用大模型替代闲聊兜底能力，而是基于语义理解需求，服务层面通过级联方式，添加如：语义规整、意图落域、知识分类、智能体、文档问答、联网搜索、兜底大模型等各种能力组合而成。

## 极速超拟人交互链路

### 3.1. 应用配置

极速超拟人交互链路是强绑定在AIUI新应用下的，链路固定开启，任何配置修改都不改变服务链路类型。

![](/media/202507/2025-07-29_201151_0867210.12699631667674804.png)

### 3.2. 流程说明

极速超拟人交互链路是通用大模型链路基础上的升级版本。服务流程链路和通用大模型基本保持一致，在功能方面针对如智能体、兜底大模型、设备人设能力方面做了更多的开放，例如智能体支持三方平台接入、支持编排和结果输出，兜底大模型支持开发者自行选配，也支持符合OpenAI API协议的三方大模型配置转发等。
