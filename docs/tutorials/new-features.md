## 前置条件

- 已创建 AIUI **极速交互** 类型应用（2025-06-12 之后创建的应用）
- 已联系讯飞商务获取对应能力授权
- 已完成 SDK 或 API 基础集成

## 你将完成的目标

通过本教程，你将了解并学会配置以下极速超拟人链路独有的新特性：

1. 流畅全双工交互 — 三层 VAD 能力，让对话自然流畅
2. 表情标签 — AI 回复附带情绪表情，增强拟人感
3. 长时记忆 — 跨会话记住用户偏好和历史
4. 声纹识别 — 识别说话人身份，实现个性化服务

::: warning 链路要求
以下所有特性仅在 **极速超拟人交互链路** 下支持。
:::

## 流畅全双工交互

AIUI 升级的全双工交互提供三层 VAD（端点检测）能力，解决传统语音交互中的「抢话」和误触发问题。

### 三层 VAD 能力

| 层级 | 名称 | 功能 | 默认状态 |
|------|------|------|---------|
| 第一层 | 语音 VAD | 通过声学特征分析过滤环境噪音，检测有效人声 | 默认开启 |
| 第二层 | 动态判停 | 根据语义动态判定是否停止收音，降低抢话频率 | 需开启 |
| 第三层 | 智能聆听 | 区分人机对话与人人对话，降低旁人说话的误触 | 需开启 |

### 各层解决的问题

**语音 VAD**：基础的人声检测和噪音过滤。当连续收到非人声信号超过尾端点时长后，自动停止收音。

::: tip 提示
语音 VAD 不能替代前端声学降噪处理。为提高识别准确率，建议对接端侧声学降噪能力。
:::

**动态判停**：解决用户思考停顿导致的提前截断问题。人在表达时天然存在停顿，简单的 VAD 可能误判为说话结束。开启动态判停后，系统根据语义判断是否继续收音。

**智能聆听**：解决多人场景下的误触发问题。系统能区分用户是在和设备说话还是在和旁人交谈，大幅降低人人对话场景的误触。

### 配置方式

在 AIUI 应用配置页面的「语义模型配置」中开启对应功能。

## 表情标签

AIUI 在 AI 回复中附带情绪表情标签（Facial Expression），让交互更有温度。当前粒度为会话级别，即每轮 AI 回复携带一个表情标签。

### 支持的表情

AIUI 当前支持 25 个表情标签，分为三大类：

**中性表情**：`neutral`（平静）、`embarrassed`（尴尬）、`surprised`（惊讶）、`thinking`（思考）、`sleepy`（困倦）、`comfort`（安慰）、`pleading`（撒娇）

**正面表情**：`happy`（开心）、`cheerful`（大笑）、`tears_of_joy`（笑哭）、`loving`（喜爱）、`cool`（酷）、`relax`（放松）、`drool`（馋）、`blow_a_kiss`（飞吻）、`congratulate`（庆祝）、`playful`（调皮）、`silly_ghost`（搞怪）

**负面表情**：`angry`（愤怒）、`sad`（悲伤）、`sigh`（叹息）、`crying`（哭泣）、`unlucky`（倒霉）、`anxiety`（焦虑）、`sorry`（抱歉）

### 应用场景

- **聊天界面**：在气泡旁展示对应 Emoji，根据情绪改变界面设计
- **虚拟形象驱动**：读取表情标签驱动 2D/3D 角色或机器人面部表情
- **情绪分析**：统计交互过程中的情绪变化趋势

### 数据格式

开启后，交互结果中会包含表情标签数据：

```json
{
  "cbm_facial_expression": {
    "compress": "raw",
    "encoding": "utf8",
    "format": "plain",
    "seq": 0,
    "status": 2,
    "text": "happy"
  }
}
```

### 配置方式

获取授权后，在应用配置中开启「表情驱动」即可。

## 长时记忆

长时记忆允许 AIUI 跨会话持久化存储用户的关键信息，打破单次对话的隔离限制。

### 核心能力

| 能力 | 说明 |
|------|------|
| 智能记忆采集 | 自动识别对话中的关键信息，进行信息抽取和压缩 |
| 动态记忆调用 | 新对话中自动检索历史记忆，注入对话上下文 |
| 全局开关控制 | 开发者可按应用场景控制是否启用 |

### 记忆特征

- 根据对话内容判定重要性，有意义的信息更不容易遗忘
- 最近内容比过往内容更详细，老旧记忆逐渐丢失细节
- 记忆冲突时新记忆覆盖旧记忆，相似信息会合并

### 记忆归属

- **开启声纹识别**：记忆在用户层级生效，不同用户的记忆隔离存储
- **未开启声纹识别**：记忆在设备层级生效，所有用户共享一份记忆

### 使用限制

| 项目 | 说明 |
|------|------|
| 对话历史回溯 | 最近 7 天 |
| 用户画像（偏好等） | 全量回溯 |
| 短时记忆 | 10 轮对话 |
| 记忆采集触发 | 满 10 轮或停止交互 10 分钟 |
| 支持的大模型 | 交互大模型 V2、儿童交互大模型 |

::: warning 重要
调用 AIUI 时需将 `new_session` 字段设置为 `false`，否则可能出现记忆丢失。详见 [极速 API 协议文档](/api-dev/ultra-chain/interact-api)。
:::

### 配置方式

授权通过后，在应用管理页面的「语义模型配置 → 个性化设置」中勾选开启。

## 声纹识别

声纹识别通过分析用户语音中的生物特征，判断「说话人是谁」，实现个性化服务。

### 核心功能

| 功能 | 说明 |
|------|------|
| 用户注册声纹 | 通过声音注册唯一标识（member_id） |
| 多份声音支持 | 每用户支持上传 3 份声音资源，提高识别率 |
| 设备级隔离 | 不同设备的声纹库互相隔离 |
| 声纹管理 | 支持增删查改操作 |
| 记忆联动 | 配合长时记忆，不同用户独立存储记忆数据 |

### 声纹信息

注册时可包含以下用户信息：

- `member_id`：用户唯一标识
- 特征描述文本
- 性别
- 年龄

### 配置方式

1. 联系商务获取授权
2. 按照 [声纹管理 API](/api-dev/ultra-chain/voiceprint-api) 注册声纹信息
3. 在应用配置的「语义模型配置 → 个性化设置」中开启声纹识别
4. 识别引擎需设置为「通用-中文-方言融合」
