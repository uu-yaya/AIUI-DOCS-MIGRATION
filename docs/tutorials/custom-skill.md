## 前置条件

- 已创建 AIUI 应用（参考 [创建应用教程](/tutorials/create-app)）
- 已登录 [AIUI 技能工作室](https://aiui.xfyun.cn/studio/skill)
- 了解基本的语义理解概念（意图、语义槽、实体）

## 你将完成的目标

通过本教程，你将学会：

1. 创建一个自定义技能并定义意图
2. 编写语料和配置语义槽
3. 创建自定义实体并绑定到语义槽
4. 配置填槽对话实现多轮交互
5. 测试并发布技能到应用中

本教程以「查询用户信息」技能为例，实现用户通过语音输入姓名、手机号和城市来查询信息的功能。

## 核心概念

| 概念 | 说明 | 示例 |
|------|------|------|
| 技能 | 处理某一类交互需求的能力单元 | 查快递、查天气、查用户信息 |
| 意图 | 用户想要完成的具体动作 | 查询用户信息 |
| 语料 | 用户表达意图的各种说法 | "我要查信息"、"帮我查一下" |
| 语义槽 | 从用户话语中提取的关键信息 | 姓名、手机号、城市 |
| 实体 | 语义槽的取值范围定义 | 手机号实体、城市实体 |

## 第一步：创建技能

1. 进入 [技能工作室](https://aiui.xfyun.cn/studio/skill)，点击 **创建技能**
2. 填写技能信息：
   - **技能名称**：查询用户信息
   - **技能标识**：`query_user_info`（创建后不可修改）
   - **技能类型**：选择适合的类型（创建后不可修改）

::: warning 注意
技能标识和技能类型在创建后无法修改，请仔细填写。
:::

## 第二步：创建意图

在技能详情页面，点击 **新建意图**：

- **意图名称**：查询用户信息
- **意图标识**：`query_info`

## 第三步：编写语料

语料是用户表达意图的各种自然语言说法。AIUI 支持两种语料编写方式：

| 类型 | 说明 | 示例 |
|------|------|------|
| 例句语料 | 纯文本，通过鼠标选字标注槽位 | 我要查张三的信息 |
| 模板语料 | 使用标签符号标注槽位 | 我要查{user_name}的信息 |

### 编写示例语料

1. 首次编写时，先 **关闭智能贴弧**（自动标注槽位），避免干扰手动标注
2. 添加至少 4 条语料，覆盖不同的表达方式：

```text
我要查信息
帮我查一下张三的信息
查询手机号18611112222的用户
我想查北京的张三
```

## 第四步：创建自定义实体

在标注语义槽之前，先创建需要的实体。

### 创建「手机号」实体

1. 进入实体管理页面，点击 **创建静态实体**
2. 设置：
   - 中文名称：`手机号`
   - 英文标识：`user_phone`
3. 添加实体词条，如：`18611111111`、`18622222222`、`18633333333`

### 创建「用户名」实体

1. 创建另一个静态实体
2. 设置：
   - 中文名称：`用户名`
   - 英文标识：`user_name`
3. 添加实体词条，如：`张三`、`李四`、`王五`

::: tip 关于实体类型
- **静态实体**：适合有限的、预定义的值列表
- **动态实体**：适合需要运行时更新的值列表
- **官方实体**：讯飞提供的通用实体，如 `IFLYTEK.ChinaCity`（中国城市）
:::

## 第五步：标注语义槽

返回意图语料编辑页面，对语料中的关键信息进行标注：

1. 鼠标框选语料中的「18611112222」，在弹出面板中选择实体 `user_phone`
2. 框选「张三」，标注为 `user_name`，选择自定义静态实体
3. 框选「合肥」，标注为 `city`，选择官方实体 `IFLYTEK.ChinaCity`

标注完成后，实体列表中会显示所有已标注的语义槽。

## 第六步：配置填槽对话

填槽对话用于在用户缺少关键信息时进行追问。

1. 在实体列表中，为每个语义槽勾选 **对话必须** 选项
2. 填写追问话术：
   - `user_name`：「你的姓名是什么」
   - `user_phone`：「你的手机号是多少」
   - `city`：「你在哪个城市」

配置完成后的交互效果：

```text
用户：我要查信息
系统：你的手机号是多少
用户：我的手机号是18611111111
系统：你在哪个城市
用户：我在北京
系统：你的姓名是什么
用户：我叫张三
系统：好的（完成意图和槽位识别）
```

## 第七步：添加技能回复

在意图配置中添加技能回复内容。如果不添加回复，模拟测试和端侧测试可能无法正常命中技能。

（可选）开启 **意图确认** 功能，用户完成所有槽位填充后会进行确认。

## 第八步：构建和测试

1. 点击右上方 **构建技能**，等待构建完成
2. 在页面右侧的测试窗口中进行文本测试
3. 验证多轮对话是否按预期工作

::: tip 语义槽标识一致性
语义槽标识会在技能业务代码中使用，请确保交互模型中的名称与业务代码保持一致。
:::

### 测试常见错误码

| 错误码 | 说明 | 处理方式 |
|--------|------|---------|
| 204 | 技能未正常返回 | 检查语料覆盖、技能后处理配置 |
| 206 | 服务调用超时 | 排查超时的服务组件 |
| 400 | 无效输入 | 检查入参格式与取值 |

## 第九步：发布技能

1. 测试通过后，点击 **发布** 按钮
2. 填写版本发布说明（方便后期问题排查）
3. 发布完成后，可在 **版本管理** 中查看版本状态

::: warning 关于技能更新
- 技能发布后，应用不会自动更新引用的技能
- 私有技能需要在应用中点击「生效」来同步更新
- 每次修改构建后，都需要重新发布才能同步到线上环境
:::

## 第十步：在应用中启用技能

1. 回到 [应用配置](https://aiui.xfyun.cn/app) 页面
2. 在 **结构化语义配置** 中添加刚发布的技能
3. 保存配置并发布应用
