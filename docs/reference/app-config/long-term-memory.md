## 背景

在普通对话系统中，模型仅依赖"上下文窗口"维持对话。一旦超出对话轮数限制或会话结束，对话信息就会丢失。

## 长时记忆介绍

**长时记忆**是指系统具备跨会话、持久化存储并检索用户关键信息的大模型原子能力。它允许 AIUI 记住用户几天、几周甚至几个月前提到过的偏好、事实和经历，并在后续对话中自然调用这些信息。

长时记忆的价值：

- **连续对话** — 打破单次会话的隔离，建立连贯的交互历史
- **个性化回复** — 基于历史数据构建用户画像，提供千人千面的对话服务
- **情感链接** — 记住用户偏好和特定要求，持续迭代交互表现，提升用户信任感

## 功能说明

目前 AIUI 已具备完整的"自动记忆闭环"，用户无需干预即可体验记忆带来的连续性。

### 智能记忆采集

系统自动识别对话中的关键信息（用户画像、对话记录），采用大模型自主决策，对原始对话记录进行信息抽取与压缩。

AIUI 记忆特征：

- 根据对话内容判定重要性，对用户有意义的信息更不容易被遗忘
- 最近的内容比过往内容更详细，老旧记忆容易丢失细节
- 记忆发生冲突时新记忆覆盖旧记忆；相似信息新旧记忆会合并

**场景示例**：用户提到"我和同学小胖经常去公园玩"，系统自动捕获隐含的社会关系"小胖是用户的同学"并存储。

### 动态记忆调用

在新对话中，系统自动检索历史记忆并注入到对话上下文。

| 特性 | 说明 |
| --- | --- |
| 记忆归属 | 打通声纹后记忆在用户层级生效，不同用户记忆隔离存储；未开启声纹时记忆在设备层级生效，所有用户共享一份记忆 |
| 检索时间范围 | 对话内容支持最近 7 天回溯；用户画像（个人信息、偏好）支持全量回溯 |
| 长短记忆配合 | 对话历史支持 10 轮（短时记忆），每满 10 轮或停止交互 10 分钟后触发一次记忆采集，转换为长时记忆 |

::: warning 重要
长短记忆需配合使用。调用 AIUI 时需将 `new_session` 字段设置为 `false`（详见[极速 API 协议文档](/api-dev/ultra-chain/interact-api)），否则可能出现记忆丢失现象。
:::

### 全局开关控制

开发者可在应用情景下配置是否为使用该情景服务的用户开启长时记忆能力。

::: warning 注意
仅支持选用**交互大模型 V2** 或**儿童交互大模型**时可调用长时记忆能力。
:::

## 使用方式

::: tip 温馨提示
长时记忆服务能力仅在 AIUI 极速超拟人交互链路下支持。
:::

1. 联系讯飞商务申请获取能力授权
2. 授权通过后，在 AIUI 极速类应用管理页面 **语义模型配置 > 个性化设置** 下勾选开启即可
