## 什么是 AIUI

AIUI 是科大讯飞于 2015 年推出的以自然语言理解为核心的全链路人机交互平台。平台整合了大模型、降噪算法、语音、图像、数字虚拟人、硬件模组等人机交互全链路软硬件核心能力，帮助 AI 硬件快速接入大模型语音交互，让设备听得清、能理解、会思考、懂执行。

AIUI 是大模型时代下重要的语音入口，不断探索人机交互的新范式，赋能新一代 AI 硬件。

## 核心能力

AIUI 整合了**语音唤醒、语音识别、大模型语义、语音合成、内容信源**等能力，并具备多模态唤醒、虚拟人驱动、多语种识别、超拟人合成、声音复刻等特性。

### 设备端能力（离线）

| 能力 | 说明 |
|---|---|
| 前端声学 | 麦克风阵列、波束形成、回声消除、混响消除、噪音抑制、声源定位 |
| 语音唤醒 | 3-6 个字的唤醒词，将设备从休眠状态切换到识别状态 |
| 离线语音识别 | 满足无网络环境下的识别需求，支持标点和实时出字 |
| 自定义命令词 | 支持开发者自定义命令词，最多 300 词 |
| 离线语义 | 支持 128 个槽位、30000 个词条，支持用户自由说 |
| 离线合成 | 支持多种语种，支持音量、语速调节 |

### 服务端能力（在线）

| 能力 | 说明 |
|---|---|
| 语音识别（IAT） | 将声音转换为文本，支持近场/远场识别、23 种方言 |
| 语义理解（NLP） | 将自然语言转化为结构化数据 |
| 语音合成（TTS） | 将文字转化为声音，支持方言、外语发音人和定制发音人 |
| 交互认知大模型 | 配置交互大模型，提高多轮交互、闲聊问答效果 |
| 信源内容 | 语义理解后的有效数据，如天气信息、音乐播放链接等 |
| 敏感词过滤 | 屏蔽涉黄、涉暴、涉政、涉恐等敏感词汇 |
| 语音翻译 | 支持英、日、韩、法、西、俄、阿拉伯等语种翻译 |

## 集成方式

AIUI 提供多种集成方式，满足不同场景需求：

- **SDK 集成**：支持 Android、iOS、Windows、Linux 平台
- **API 集成**：提供 WebSocket API 接口
- **硬件模组**：开箱即用的硬件方案

## 三种服务链路

AIUI 平台针对语义交互场景提供三种服务链路：

| 链路 | 说明 | 适用场景 |
|---|---|---|
| [传统语义交互链路](/platform-service/service-chain#传统语义交互链路) | 以传构化语义理解能力为核心，采用语法句式匹配方案 | 需要精确结构化结果的设备控制场景 |
| [通用大模型交互链路](/platform-service/service-chain#通用大模型交互链路) | 基于大模型的级联能力组合，支持智能体、文档问答、联网搜索等 | 需要更灵活自然语言理解的通用场景 |
| [极速超拟人交互链路](/platform-service/service-chain#极速超拟人交互链路) | 通用大模型链路的升级版，支持三方大模型配置、智能体编排等 | 追求极致交互体验的新项目 |

::: tip 如何选择
如果你是新开发者，建议从**极速超拟人交互链路**开始。2025 年 6 月 12 日后创建的新应用默认使用此链路。详细对比请参阅 [选择你的接入路径](./choose-your-path)。
:::

## 行业应用

AIUI 已整合的行业方案包括：儿童玩具、儿童教育、语音点歌、智能投影、智能鼠标、多语种 AI 透明屏、移动数字人、大屏调度、服务机器人、营销机器人、智慧轨道交通等。
