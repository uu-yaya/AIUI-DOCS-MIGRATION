## 如何使用本指南

本章将原 AIUI、评估板、动态实体三篇 FAQ 按**症状**重新组织。请根据你遇到的现象选择对应章节，每个条目包含：

1. **症状描述** — 你看到了什么
2. **可能原因** — 为什么会这样
3. **解决步骤** — 怎么修复

## 连接与网络

### 评估板提示"网络有点问题，我去休息了"

**症状**：评估板语音交互后播报网络异常提示，无法获取结果。

**可能原因**：超时错误码 `10120`——语音上传后，听写或语义结果未在 `result_timeout` 规定时间内返回。常见于网络不稳定或服务器繁忙。

**解决步骤**：

1. 在 AIUI 模块上执行 `ping aiui.openspeech.cn`，确认网络连通性。
2. 将开启日志的 `msc.cfg` 放入 `/sdcard/msc/` 目录。
3. 复现问题后，将 `/sdcard/msc/` 下的日志文件导出，发送给技术支持分析。

### 在国外使用 AIUI 响应慢

**症状**：海外网络环境下交互延迟明显高于国内。

**可能原因**：AIUI 服务部署在国内，跨境网络延迟较高。

**解决步骤**：

- AIUI 支持国外使用，但响应会比国内慢。如延迟不可接受，建议优化网络链路（如使用专线或加速服务）。

## 语音识别

### 专业领域词汇识别率低

**症状**：特定产品名、术语等频繁被识别为同音或近音词（如"燕京啤酒"被识别为"眼睛啤酒"）。

**可能原因**：通用语言模型未覆盖该领域词汇。

**解决步骤**：

1. 在 AIUI 控制台 → 我的应用中上传热词。
2. 应用级热词生效时间 < 60 分钟，用户级热词生效时间 < 10 分钟，"所见即可说"资源立即生效。

### 英文识别"优先阿拉伯数字"无效

**症状**：平台勾选了数字优先选项，英文识别时仍输出英文数字。

**可能原因**：英文识别暂不支持平台端的数字优先配置。

**解决步骤**：

- 目前无平台侧解决方案，需在客户端自行做数字格式转换。

### 不支持 8k 音频

**症状**：写入 8k 采样率音频后识别失败或结果异常。

**可能原因**：AIUI SDK 仅支持 16k/16bit 音频。

**解决步骤**：

- SDK 端：必须使用 16k/16bit 单声道音频。
- WebAPI 端：支持 8k 和 16k 音频。

### 方言和多语种识别

**症状**：需要识别方言或非中文语种，但结果不理想。

**可能原因**：AIUI 目前支持中文普通话、粤语、四川话和英文。其他语种需商务申请。

**解决步骤**：

1. 确认目标语种是否在支持范围内。
2. 如需其他语种，提交工单进行商务申请。
3. 英文语义理解不支持官方开放技能（音乐、天气等），但可通过自定义技能实现。英文技能不支持模糊匹配。

## 语音合成

### 替换发音人后闲聊回答无声音

**症状**：切换发音人后，闲聊技能下部分回答无法播报，但音乐、天气等技能正常。

**可能原因**：闲聊回答包含情感标签，目前仅"萌萌"发音人支持情感合成。其他发音人遇到情感标签时无法合成。

**解决步骤**：

1. 切换回"萌萌"发音人验证问题。
2. 如需使用其他发音人，需在业务逻辑中过滤情感标签后再合成。

### 评估板唤醒后仍为萌萌的声音

**症状**：已替换发音人，交互时声音已变，但唤醒提示音不变。

**可能原因**：唤醒提示音为预录制音频，非实时合成。

**解决步骤**：

- 修改 AIUIProductDemo 源码中唤醒播报对应位置，替换为自定义音频文件。

### TTS 合成音频保存方法

**症状**：需要将语音合成结果保存为音频文件。

**可能原因**：默认不自动保存合成音频。

**解决步骤**：

集成 MSC 的 `SpeechSynthesizer` 时设置以下参数：

```java
mTTS.setParams(SpeechConstant.AUDIO_FORMAT, "wav");
mTTS.setParams(SpeechConstant.TTS_AUDIO_PATH, "mnt/sdcard/tts/xxx.wav");
```

## 交互模式

### Continuous 和 Oneshot 模式的区别

**症状**：不确定应选择哪种交互模式。

**说明**：

| 模式 | 特点 | 适用场景 |
|---|---|---|
| Continuous（全双工） | 一次唤醒，多次交互 | 配合麦克风阵列的智能硬件 |
| Oneshot（单轮交互） | 一次唤醒，一次交互 | 按住按钮说话的移动端 |

- 两种模式均支持每次最长 1 分钟的持续音频写入，不可无限写入。

### 不支持一直保持唤醒状态

**症状**：希望 AIUI 持续处于唤醒状态，但一段时间后自动休眠。

**可能原因**：AIUI 设计为超时自动休眠。距最后一次有效语义结果（`rc` 不为 4）超过设定时间后进入休眠。

**解决步骤**：

- 超时休眠时间可配置，范围 `[10000, 180000)` 毫秒（10 秒到 3 分钟）。
- 配置方式参考 AIUI 参数设置文档。

### 多轮对话退出与历史清除

**症状**：不清楚多轮对话何时结束、如何清除交互历史。

**说明**：

- 服务端保存交互历史最长 120 秒，超过 5 轮后自动清空。
- 默认模式（`auto`）：休眠后唤醒清除历史；`STATE_WORKING` 状态下唤醒不清除。
- 可配置为 `user` 模式，通过 `CMD_CLEAN_DIALOG_HISTORY` 手动清除。

### 移动端合成时识别到自己的声音

**症状**：Continuous 模式下，TTS 播放的声音被麦克风捕获，导致答非所问。

**可能原因**：移动端无麦克风阵列，缺少降噪和回声消除能力。

**解决步骤**：

1. 切换为 Oneshot 交互模式（一次唤醒一次交互）。
2. 或采用带麦克风阵列的智能硬件方案。

## 评估板

### APPID 未配置

**症状**：替换配置文件后开机提示"APPID 未配置"。

**可能原因**：SD 卡上的 `aiui.cfg` 不是 UTF-8 无 BOM 的 JSON 格式，被系统自动清空。

**解决步骤**：

1. 用文本编辑器打开 `aiui.cfg`，确认编码为 UTF-8（无 BOM）。
2. 确认 JSON 格式正确（可用 JSON 校验工具检查）。

### 配置后唤醒和交互无声音

**症状**：评估板配置完成后唤醒和交互均无声音，但手机端可看到交互结果。

**可能原因**：手机端同步配置时未勾选"是否启动后处理"。

**解决步骤**：

1. 在手机端勾选"是否启动后处理"。
2. 同步配置到评估板。
3. 如仍无声音，联系技术支持进一步排查。

### 错误码 10407：APPID 与 KEY 不匹配

**症状**：手机端配置评估板时显示 `10407` 错误。

**可能原因**：配置的 AppID 与 APIKey 不匹配。

**解决步骤**：

1. 登录 AIUI 控制台，确认应用的 AppID 和 APIKey 配对关系。
2. 重新填写正确的配置信息并同步。

### 授权不足，服务停止

**症状**：评估板报"应用授权不足，服务停止"。

**可能原因**：每个 AppID 默认开通 5 个装机量授权，超出后触发此限制。

**解决步骤**：

- 联系 [msp_business@iflytek.com](mailto:msp_business@iflytek.com) 申请更多授权。

### USB 连接电脑无反应

**症状**：评估板通过 USB 连接电脑后无任何反应。

**可能原因**：未连接 Debug 口，或数据线不支持数据传输。

**解决步骤**：

1. 确认连接的是评估板的 **Debug 口**。
2. 更换数据线重试。
3. 如仍无法连接，使用 PC 版手机助手安装驱动后重试。

### 串口合成状态获取

**症状**：通过串口控制评估板语音合成，无法获取合成进度。

**可能原因**：核心板仅返回合成开始和结束状态，不反馈中间进度。

**解决步骤**：

- 如需合成进度，通过自定义串口消息修改 AIUIProductDemo 源代码实现。

## 动态实体

### 上传资源失败，返回 -1

**症状**：客户端上传"所见即可说"资源失败，返回值为 `-1`。

**可能原因**：资源名、命名空间或技能名配置错误。

**解决步骤**：

1. 确认资源名是 AIUI 平台上填写的资源名（非自定义名称）。
2. 确认资源数据中包含命名空间。
3. 实体信息使用英文符号，不能有空格。格式示例：
   ```json
   {"name":"微信","alias":"wechat","extra":"xxx"}
   ```
4. 确认技能名填写正确。

### 上传成功但 slots 为空

**症状**：动态实体上传成功，但语音交互时 `slots` 始终为空。

**可能原因**：数据未生效、输入方式不正确或解析代码有误。

**解决步骤**：

1. 等待 10-60 分钟，动态实体需要时间生效。
2. 确认使用**语音输入**测试（文本输入不触发动态实体）。
3. 检查解析前的原始数据，排除解析代码错误。
4. 检查生效代码逻辑是否正确。

### 数据上传是否增量更新

**症状**：多次上传动态实体数据后，之前的数据消失。

**可能原因**：动态实体上传为全量覆盖，非增量更新。

**解决步骤**：

- 每次上传需包含全部数据，新数据会完全覆盖旧数据。

### 多个别名字段的写法

**症状**：不确定如何在动态实体中填写多个别名。

**解决步骤**：

两种方式：

1. 用 `|` 分隔：`{"name":"银行卡查询","alias":"查询银行卡|行卡查询"}`
2. 使用 `alias1`、`alias2` 等编号字段，最多 10 个，英文逗号隔开。

### 数据上限

**症状**：上传大量实体数据时失败。

**可能原因**：超出大小限制。

**解决步骤**：

- 动态实体单次上传上限为 10KB（Base64 编码后的数据大小）。

## SDK 集成

### ARM/MIPS 架构如何获取 SDK

**症状**：目标设备为 ARM 或 MIPS 架构，平台无对应 SDK 下载。

**可能原因**：开发板编译器种类繁多，讯飞无法预编译全部架构。

**解决步骤**：

1. 每个客户可获得 2 次免费交叉编译服务。
2. 超出次数需联系 AIUI 平台商务。
3. 邮件联系 [aiui_support@iflytek.com](mailto:aiui_support@iflytek.com) 或加入开发者 QQ 群申请。

### Linux/Windows 平台录音

**症状**：`CMD_START_RECORD` 仅支持 Android，Linux/Windows 上无法使用内置录音。

**可能原因**：不同发行版音频接口差异大，SDK 无法统一兼容。

**解决步骤**：

- 采用外部录音方案，自行采集 16k/16bit 音频，通过 `CMD_WRITE` 写入 AIUI。
- 讯飞提供了 Ubuntu（ALSA）和 Win7（Wavein）的录音源码示例。

### 蓝牙耳机录音无数据

**症状**：Android 手机连接蓝牙耳机后，音频无法传入 AIUI。

**可能原因**：Android 系统录音通道未自动切换到 SCO 模式。A2DP 仅支持播放，SCO 支持双向音频但默认关闭。

**解决步骤**：

录音前开启 SCO：

```java
AudioManager mAudioManager = (AudioManager) getSystemService(Context.AUDIO_SERVICE);
mAudioManager.setBluetoothScoOn(true);
mAudioManager.startBluetoothSco();
```

录音结束后关闭 SCO：

```java
mAudioManager.setBluetoothScoOn(false);
mAudioManager.stopBluetoothSco();
```

### iOS 编译报 "include string file not found"

**症状**：Xcode 编译 AIUI 项目时报找不到 `include <string>`。

**可能原因**：`AIUIProcessor` 文件使用了 C++ 特性，但文件后缀为 `.m`。

**解决步骤**：

- 将文件后缀名从 `.m` 改为 `.mm`，启用 Objective-C++ 编译。

### AIUI 唤醒功能是否需要单独集成 MSC

**症状**：不确定是否需要额外集成 MSC 的 `VoiceWakeUper`。

**说明**：

- 不需要。AIUI SDK 已内部封装唤醒调用逻辑。
- 只需将唤醒词文件和 SDK 库（`libmsc.so` 或 `msc.dll`）放到指定位置并配置即可。

## 平台功能

### 交互次数计量方式

**症状**：不清楚一次交互如何计费。

**说明**：

- 以单次会话发送结束帧后、云端有正常结果下发算一次交互（异常不消耗次数）。
- 一次音频请求下发的识别、语义、合成结果合计为一次交互。
- 会话唯一标记为语义结果中的 `sid` 字段。

### 收费标准

**症状**：不清楚 AIUI 的费用。

**说明**：

- 每个应用包含 500 次/天的测试量。
- 正式量产需联系 [aiui_support@iflytek.com](mailto:aiui_support@iflytek.com) 咨询合作。

### 快速体验界面不支持后处理调试

**症状**：应用配置了后处理，但在平台"快速体验"界面测试无反应。

**可能原因**：快速体验功能不支持后处理业务调试。

**解决步骤**：

- 使用客户端 SDK 进行后处理功能调试。

### 应用和技能不支持删除

**症状**：想删除已创建的应用或技能，找不到删除入口。

**说明**：

- 目前应用和技能暂不支持删除。后续功能以平台通知为准。

### 自定义问答库不支持英文和 API 导入

**症状**：需要英文问答或批量导入问答数据。

**说明**：

- 问答库不支持英文。
- 不支持 API 导入，仅支持在平台页面手动配置。

## 错误码速查

| 错误码 | 含义 | 处理方式 |
|---|---|---|
| `10120` | 结果超时 | 检查网络连通性；导出日志联系技术支持 |
| `10407` | AppID 与 APIKey 不匹配 | 核对控制台中的 AppID/APIKey 配对 |
| `RC3` | 信源访问异常 | 检查 URL 可达性、播放格式兼容性、HTTPS 证书是否过期 |
| `-1`（动态实体上传） | 资源配置错误 | 检查资源名、命名空间、技能名 |
