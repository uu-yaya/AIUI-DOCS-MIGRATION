## 平台与架构

### AIUI

科大讯飞推出的以自然语言理解为核心的全链路人机交互平台，整合了语音唤醒、语音识别、语义理解、语音合成、内容信源等能力。

### 服务链路

AIUI 平台提供的语义交互处理流程，目前有三种：传统语义交互链路、通用大模型交互链路、极速超拟人交互链路。详见 [服务链路介绍](/platform-service/service-chain)。

### AppID

AIUI 应用的唯一标识符，在创建应用时由平台自动分配。

### APIKey

用于 API 接口认证的密钥，与 AppID 配合使用。

### APISecret

用于 API 接口签名的密钥，老应用中 2022-06-01 之前创建的应用可能不包含此字段。

## 语音技术

### IAT（语音识别）

Intelligent Automatic Transcription，将语音信号转换为文本的技术。AIUI 支持近场识别（距离 < 1m）和远场识别（距离 1~5m），以及 23 种方言。

### TTS（语音合成）

Text-To-Speech，将文本转换为语音的技术。支持多种发音人、方言和外语，也支持定制发音人。

### VAD（端点检测）

Voice Activity Detection，用于判断一句话的开始（前端点 BOS）与结束（后端点 EOS）。

### NLP（语义理解）

Natural Language Processing，将自然语言转化为结构化数据。例如将"合肥天气"处理为包含城市和时间的 JSON 结构。

### 语音唤醒

通过 3-6 个字的唤醒词（如"小飞小飞"），将设备从休眠状态切换到识别状态的技术。

### 前端声学

包括麦克风阵列、波束形成、回声消除、混响消除、噪音抑制、声源定位等算法，用于提高语音唤醒率和识别率。

## 交互模式

### Oneshot（单轮交互）

一次唤醒对应一次交互。适用于手机 APP 或语音遥控器等单麦克风设备，需要用户或程序主动触发录音。

### Continuous（全双工）

一次唤醒后可连续交互，设备喇叭发声的同时录音，无需唤醒词打断。要求设备具备良好的回声消除效果且环境较安静。

### 免唤醒交互

无需唤醒词即可直接说话控制设备，支持离线使用。

### 多模态交互

融合声纹识别、手势识别、唇形检测、虚拟人形象等多种 AI 技术的交互方式。

## 技能与语义

### 技能

AIUI 系统中的特定功能模块，用于完成用户的特定任务或提供特定服务。技能分为商店技能（平台预置）和自定义技能（开发者创建）。

### 意图

用户的目的或需求。每个意图代表一个或一组功能，分为**入口意图**（不需要历史信息即可响应）和**对话意图**（需要上下文关联才能命中）。

### 语料

表达用户意图的自然语言样本集合。分为**模版语料**（由词槽和文本构成，如 `{time}天气`）和**例句语料**（纯文本）。两者不可在同一意图中混用。

### 实体

同一类型词汇的集合（词库）。例如 `{city}` 实体包含北京、上海、广州等。分为自定义实体和官方实体。

### 静态实体

不常更改的自定义实体数据，开发者直接在平台上修改更新，实时永久生效。

### 动态实体

根据端侧环境或用户不同而动态变更的实体数据。生效范围分为应用级、用户级和自定义级，数据上传为覆盖更新。

### 所见即可说

一种特殊的动态实体，数据上传后仅一次交互后即失效，适用于界面动态更新的临时数据场景。

### 词槽

语料模板中用 `{}` 标记的变量位置，每个词槽可绑定一个实体。词槽命中的实体信息在语义结果的 `slots` 字段中返回。

### 填槽对话

系统通过多轮对话主动询问用户以获取完成任务所需关键信息的交互方式。当必要槽位缺失时，系统会按预设的追问话术逐一询问。

### 模糊匹配

当用户输入存在拼写错误或表述不完整时，系统通过通配实体等方式仍能识别其真实意图的技术。

## SDK 与开发

### AIUIAgent

SDK 中与 AIUI 服务交互的核心对象（桥梁）。通过 `createAgent` 创建实例，通过 `sendMessage` 发送消息，通过 `AIUIListener` 接收回调。

### aiui.cfg

AIUI SDK 的配置文件，包含 `login.appid`（应用 ID）、`global.scene`（场景名称）、`aiui_ver`（链路版本号）等关键参数。

### aiui_ver

`aiui.cfg` 中的链路版本参数。取值 `1` 对应传统语义链路，`2` 对应通用大模型链路，`3` 对应极速超拟人链路。

### 场景（Scene）

AIUI 应用的配置环境。区分沙盒场景（测试用，`main_box`）和线上场景（`main`），发布后沙盒配置同步到线上。

## 信源与内容

### 信源

语义理解后返回的有效数据内容，例如天气信息、音乐播放链接等。AIUI 技能商店中多数技能包含信源内容。

### 后处理

AIUI 云服务将语义结果转发给开发者自定义的后处理服务进行业务处理，然后将结果返回给 AIUI 进行后续流程（如语音合成）。
