## 4.接口连接指南

**4.1将麦克风阵列连接至 RK3328 AIUI评估板开发套件**

麦克风阵列连线采用2\*8-2.0 牛角排线，将插头按照缺口方向插入插座。

![](/media/202309/1693904775.075794.png)

图 2 RK3328 AIUI评估板开发套件麦克风阵列接线图

**4.2将扬声器连接至RK3328 AIUI评估板开发套件**

扬声器接口采用 PH2.54-4P 座，将插头按照缺口方向插入插座。 如更换扬声器，请参照接口线序标识，L-代表左声道负极，L+代表左声道正极，R-代表右声道负极，R+代表右声道正极。如果采用外置功放，此接口不接。

![](/media/202309/1693904775.080359.png)

图 3 RK3328片评估板开发套件扬声器接线图示

**4.3将TTL串口连接到上位机。**

TTL 串口采用 PH2.0-3P 插座， 将插头按照缺口方向插入插座。 为3.3VTTL 电平， 可采用USB 转 TTL 与上位机连接。 请参照 RK3328 评估板开发套件 的 TTL 串口接口线序标识， TX为数据发送， RX为数据接收， GND为供地。

![](/media/202309/1693904775.1202416.png)

图 4 RK3328 AIUI评估板开发套件串口接线图示

**4.4将电源、HDMI线，连接到 RK3328 AIUI评估板开发套件**

电源接口采用 DC5.5-2.1 插座或 PH2.54-2P 插座， 将插头按照缺口方向插入插座。 电源电压为 DC12V， 使用板载功放时， 提供电流不低于 3A。不使用板载功放时，提供电流不低于1A，请参照 RK3328 评估板开发套件的电源接口线序标识， DC12V 为电源正， GND为电源负。

## 5.使用方法

本章介绍两种使用方式，如果自带功放满足扬声器声音大小的需求，则使用板载功放。若板载功放无法满足需求，则可以使用外接功放。

## 5.1使用板载功放

使用RK3328 AIUI评估板开发套件板载功放时，将2个≤10W 4欧喇叭连接到RK3328 AIUI评估板开发套件上，并将串口按照线序连接到客户控制板上。

![](/media/202309/1693904775.1435633.png)

![](/media/202309/1693904775.1630986.png)

## 5.2使用外置功放

## 6.快速体验指南

**第一步：将所有连线接按照第三、四章连接好，将HDMI连接到电视机，在PC机端打开sscom 串口工具。**

串口工具， 请在提供的工具包内查找。

**第二步：配置串口工具设置，并打开串口。**

选择识别出来的USB转TTL 串口号， 如果USB转TTL没有驱动， 请在提供的工具包内查找并安装。设置波特率：1500000；数据位8；奇偶校验：无；停止位：1 流控制： 无。

**第三步：RK3328 AIUI评估板开发套件接通电源，查看串口启动信息。**

接通电源后，大概1分钟左右，串口会收到启动信息。

**第四步： 体验语言唤醒、声源定位、噪声抑制、回声消除。**

- **唤醒:**

启动评估板开发套件，当听到“叮”的一声提示音之后即可进行唤醒，默认唤醒词“小飞小飞”

- **交互**

设备唤醒后支持持续的交互，用户可以连续的对话，并且在音频播放的过程中可打断，继续发出指令；

持续交互的过程中，支持说话人或其他人再次唤醒，再次唤醒将会清空之前

的音频和交互结果。

全双工交互过程中，识别引擎会对输入的无效语音和噪音进行过滤和拒识。

- **休眠**

当 AIUI 发现超过 1 分钟（可设置）没有有效交互结果时，即会自动进入休眠状态。自动休眠时，正在播放的歌曲不会停止。喊出下方指令可以主动使 AIUI 进入休眠：跪安吧、退下吧、走开、安静会儿、休眠、关机、睡觉吧、 你去休息吧。

进入休眠后，唤醒时会清除上一轮的交互历史。

## 7.替换唤醒词

RK3328评估板目前默认环形6麦算法集成的是AIUI平台最新优化唤醒响应时间的降噪算法，该版本唤醒词资源打包工具暂未上线平台，所以如需更换唤醒词可以联系讯飞线下支持获取（邮件发送 [aiui\_support@iflytek.com](mailto:aiui_support@iflytek.com) 或联系讯飞商务同事申请获取），线下获取最新唤醒词资源后，查看资源文件，一般有三个文件：

- keyword.bin
- mlp.bin
- filler.irf

主要需要使用 keyword.bin 资源，下面就替换流程做简单说明：
1、usb连接到PC端
2、使用adb命令操作，查看设备具体加载资源存放路径（默认环形6麦）

```bash
adb shell
cd /sdcard/AIUI/assets/vtn/config
cat vtn_mic6_circle.ini
```

![](/media/202411/2024-11-21_175750_4933040.21462302467886696.png)

3、在根据所在文件目录和名称直接重命名覆盖替换
建议：对原唤醒词资源文件进行备份，避免操作失误导致异常问题无法恢复。

## 8.发音人和音频替换

在3328评估板 /sdcard/ProductDemo/wav/ 文件夹下，保存了内置提示音频，具体说明如下：
![](/media/202412/2024-12-17_103843_8114410.7318005693452063.png)

|  |  |  |
| --- | --- | --- |
| **音频名称** | **内容** | **音频说明** |
| xxx\_greeting\_ss.wav | 啥事 | 唤醒提示音频1 |
| xxx\_greeting\_wz.wav | 我在 | 唤醒提示音频2 |
| xxx\_greeting\_zai.wav | 在 | 唤醒提示音频3 |
| xxx\_greeting\_zaine.wav | 在呢 | 唤醒提示音频4 |
| xxx\_network\_error.wav | 网络有点问题，我去休息了，请稍后再试 | 网络异常提示音频 |
| xxx\_no\_network.wav | 网络未连接，请连接网络 | 网络未连接提示音频 |
| xxx\_wrong\_appid.mp3 | appid配置不成功，请先在手机端配置 | appid配置提示音频 |
| alarm.mp3 | \ | 闹钟提示音频 |
| search\_result.mp3 | \ | 查询提示音频 |
| start\_success.mp3 | \ | 开机和配网成功提示音频 |
| tone\_goodbye.mp3 | \ | 休眠提示音频 |
| volume.wav | \ | 音量操作提示音频 |

### 注意：

音频文件名称中的 xxx 代表发音人名称，对应aiui.cfg中voice\_name 取值。当发音人名称所在音频不存在时，默认加载 x4\_lingxiaoying\_em\_v2 发音人对应的相关提示音频

修改方式：
1、本地按照音频内容制作相关音频文件，并根据需要替换的发音人名称重新命名，最后push到 /sdcard/ProductDemo/wav 目录下。
例如唤醒提示音频1，需要修改的发音人为：x\_qige，则音频命名为 x\_qige\_greeting\_ss.wav
2、从 /sdcard/AIUI/cfg 目录下pull 出 aiui.cfg，修改 TTS（语音合成）.voice\_name 取值为需要的发音人名称，如 x\_qige。
修改完成后push aiui.cfg到原目录下进行替换
![](/media/202412/2024-12-17_111330_7281750.7374590356767292.png)

3、adb reboot 重启设备即可
